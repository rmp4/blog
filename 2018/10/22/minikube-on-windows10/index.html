Requirements

在windows command prompt底下輸入

systeminfo

如果看到以下輸出，則Windows支持虛擬化。

Hyper-V Requirements:     VM Monitor Mode Extensions: Yes
                      Virtualization Enabled In Firmware: Yes
                      Second Level Address Translation: Yes
                      Data Execution Prevention Available: Yes

如果有看到下面顯示的表示你已經安裝hyper-v

Hyper-V Requirements:     A hypervisor has been detected. Features required for Hyper-V will not be displayed.

啟動指令

首先minikube start會默認使用VirtualBox，但是我們用的是Hyper-V，所以需要加參數指定：

minikube start --vm-driver=hyperv

還記得我們之前創建的虛擬網絡交換機嗎？這時候就需要用到它了。

minikube start --vm-driver=hyperv --hyperv-virtual-switch=k8svswitch

默認不顯示錯誤信息，為了方便排查啟動中的問題，我們還是開啟日誌輸出。

minikube start --vm-driver=hyperv --hyperv-virtual-switch=k8svswitch --v=3 --alsologtostderr

完成後會看到

* Done! kubectl is now configured to use "minikube"

便可以使用kubectl執行指令如

PS C:\WINDOWS\system32> kubectl.exe get node
NAME       STATUS    ROLES     AGE       VERSION
minikube   Ready     master    17m       v1.15.0
